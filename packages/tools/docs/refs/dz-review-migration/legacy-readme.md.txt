# DZ Markdown Review Syntax

VS Code syntax highlighting for Markdown review loops between an agent and a
human reviewer, plus Obsidian-flavoured Markdown and custom review annotations.

This extension injects into VS Code's built-in Markdown language. It does not
contribute a separate language mode, so it can compose with other Markdown
syntax extensions.

It also highlights Obsidian comments, wikilinks, embeds, callouts, tags, block
references, `==highlights==`, and custom review additions, deletions,
substitutions, highlights, and comments.

This repository is also an agent plugin source:

- `.codex-plugin/plugin.json` exposes it as a Codex plugin.
- `.claude-plugin/plugin.json` exposes it as a Claude Code plugin.
- `skills/markdown-review-workflow` teaches agents how to create, answer,
  resolve, and clean inline Markdown review conversations.

The default review syntax stays valid Markdown by using one HTML comment block
per conversation. Each message starts on its own line:

```markdown
The generated text stays normal Markdown.
<!--
@agent I chose this wording because it matches the glossary.
@me Can we make this less categorical?
@agent Done. I softened the wording and kept the glossary term.
-->
```

The opening `<!--` may appear after paragraph text, a list item, or a heading.
Only comments that contain review lines are treated as active conversations.

The extension also supports a custom conversation block:

```markdown
{??
@agent I chose this wording because it matches the glossary.
@me Can we make this less categorical?
??}
```

Set `dzMdReview.commentSyntax` to `custom` to create new
conversations with `{?? ... ??}`. The default is `html`. Existing conversations
in HTML and `{?? ... ??}` syntaxes are recognized.

Set `dzMdReview.timestampFormat` to `compact`, `iso`, or `none` to choose the
timestamp behavior used when the extension creates new conversation messages and
review annotations. The default is `compact`. Use `none` for timestamp-free
workshop notes from shortcuts such as `cmd+alt+enter` and `cmd+alt+k d`.
The explicit `cmd+alt+k t` command still adds a compact timestamp when this
setting is `none`.

Canonical roles:

- `@agent`: agent rationale or note about generated content.
- `@me`: human review comment.

Older notes that use a colon after the role marker are still recognized, but
commands generated by the extension use the colonless form: `@agent Note` and
`@me Note`.

Conversation messages may include a timestamp immediately after the role marker
with a `%` prefix. The extension adds compact timestamps by default:

```markdown
<!--
@agent%1WzvP91W I chose this wording because it matches the glossary.
@me%2026-06-16T17:35:35+0200 Can we make this less categorical?
-->
```

The ISO form is readable and explicit. The compact form is shorter, sortable,
and can be decoded by the extension and CLI. Hovering a compact timestamp in VS
Code shows its ISO timestamp.

For fast manual review, a line that starts with a bare `@` inside the block is
highlighted as a temporary human comment:

```markdown
{??
@ This is a quick review note while reading.
??}
```

The agent should normalize these quick notes when processing the review:

```markdown
{??
@me This is a quick review note while reading.
??}
```

When a review is resolved, the agent should replace the conversation block with
a validated note. The default validated note is a plain Markdown HTML comment:

```markdown
<!-- I chose this wording because it matches the glossary. -->
```

For Obsidian notes, the agent may use an Obsidian comment instead:

```markdown
%%
I chose this wording because it matches the glossary.
%%
```

Validated notes no longer contain `@agent` or `@me`, so this extension ignores
them when navigating or deleting active review conversations. Obsidian comments
are highlighted by the bundled Obsidian syntax injection.

Conversation blocks should not be nested, so replies should be new `@agent` or
`@me` lines inside the same block. The document remains readable in source
form, while the extension can color each role differently.

The extension contributes default token colors for the two review roles: agent
notes and human comments. They remain normal TextMate scopes, so local VS Code
settings or themes can override them.

For Catppuccin Latte, it also contributes default colors for the bundled
Obsidian and custom review annotation scopes.

Active review conversations also receive editor decorations:

- a subtle background on the conversation content;
- a marker in the overview ruler;
- a 💬 gutter marker on the line where the conversation starts.

Commands:

- `Markdown Review: Add Human Comment` inserts an indented `@me` reply before
  the closing marker when used on the last non-empty content line, or on the
  closing marker line, of a multiline conversation. On compact inline notes, it
  expands the note to multiline and adds `@me`. Elsewhere, it keeps the normal
  new-line behavior. This command has no default keybinding.
- `Markdown Review: Add Quick Comment` adds a bare human reply. Outside a
  conversation, it creates a compact `<!-- @  -->` note, with `{==...==}`
  around a selection when present. Inside an inline conversation, it appends
  `@ ` before the closing marker. Inside a multiline conversation, it inserts
  a new `@ ` line before the closing marker.
- `Markdown Review: Add OK Reply` adds a trailing `@me ok` reply without
  toggling it off when it already exists.
- `Markdown Review: Remove OK Reply` removes a trailing `@me ok` reply without
  adding one when it is absent.
- `Markdown Review: Create Multiline Quick Comment or Toggle Conversation
  Layout` creates multiline `<!-- ... -->` quick comments outside
  conversations. With a selection it writes
  `{==selection==}<!--\n@ \n-->`. Inside conversations, it toggles between
  inline and multiline forms. When compacting, it joins distinct review
  messages with spaces but preserves continuation lines inside each message.
- `Markdown Review: Create Review Discussion` creates a compact
  `{?? @me  ??}` discussion note regardless of `dzMdReview.commentSyntax`.
- `Markdown Review: Add Review Addition` wraps the selection with
  `{++...++}`, or inserts `{++++}` with the cursor inside.
- `Markdown Review: Add Review Deletion` wraps the selection with
  `{--...--}`, or inserts `{----}` with the cursor inside.
- `Markdown Review: Add Review Substitution` wraps the selection with
  `{~~old~>new~~}` and places the cursor on the replacement side.
- `Markdown Review: Add Review Highlight` wraps the selection with
  `{==...==}`, or inserts `{====}` with the cursor inside.
- `Markdown Review: Add Review Comment` wraps the selection with
  `{>>...<<}`, or inserts `{>><<}` with the cursor inside.
- `Markdown Review: Add Timestamp to Review Element` adds timestamp metadata to
  the conversation message or review annotation at the cursor. It leaves
  already timestamped elements unchanged.
- `Markdown Review: Cancel Review Annotation` cancels the annotation at
  the cursor: additions and comments are removed, deletions and substitutions
  keep the old text, highlights keep their text, and discussions are removed.
- `Markdown Review: Apply Review Annotation` applies the annotation at
  the cursor: additions keep the new text, deletions and comments are removed,
  substitutions keep the replacement, highlights keep their text, and
  discussions are removed.
- `Markdown Review: Delete Conversation` removes the current review
  conversation.
- `Markdown Review: Go to Next Review Block` moves to the next custom review
  annotation or review conversation.
- `Markdown Review: Go to Previous Review Block` moves to the previous
  custom review annotation or review conversation.
- `Markdown Review: Go to Next Conversation` moves to the next review
  conversation.
- `Markdown Review: Go to Previous Conversation` moves to the previous review
  conversation.
- `Markdown Review: Toggle Review Mode` enables a modal review pass for the
  active Markdown editor. While it is active, the status bar shows `Review` and
  single-key review shortcuts are enabled. Use `escape` to leave the mode.

Default macOS shortcuts:

- `cmd+alt+k m`: toggle review mode.
- `cmd+alt+enter`: add quick `@ ` comment.
- `cmd+alt+shift+enter`: create a multiline quick comment or toggle
  conversation layout.
- `cmd+alt+k a`: add custom review addition.
- `cmd+alt+k s`: add custom review deletion.
- `cmd+alt+k r`: add custom review substitution.
- `cmd+alt+k h`: add custom review highlight.
- `cmd+alt+k c`: add custom review comment.
- `cmd+alt+k d`: create custom review discussion note.
- `cmd+alt+k t`: add a timestamp to the review element at the cursor.
- `cmd+alt+k o`: add `@me ok`.
- `cmd+alt+k shift+o`: remove trailing `@me ok`.
- `cmd+alt+k x`: cancel custom review annotation.
- `cmd+alt+k shift+x`: apply custom review annotation.
- `cmd+alt+k n`: next custom review annotation or review conversation.
- `cmd+alt+k shift+n`: previous custom review annotation or review conversation.
- `cmd+alt+k p`: next open or wip conversation.
- `cmd+alt+k shift+p`: previous open or wip conversation.
- `cmd+alt+backspace`: delete conversation.
- `cmd+ctrl+down`: next conversation.
- `cmd+ctrl+up`: previous conversation.
- `cmd+ctrl+alt+down`: next open or wip conversation.
- `cmd+ctrl+alt+up`: previous open or wip conversation.

Review mode shortcuts:

- `n`: next custom review annotation or review conversation.
- `shift+n`: previous custom review annotation or review conversation.
- `p`: next open or wip conversation.
- `shift+p`: previous open or wip conversation.
- `x`: cancel custom review annotation.
- `shift+x`: apply custom review annotation.
- `o`: add `@me ok`.
- `@`: add quick `@ ` comment.
- `escape`: exit review mode.

## CLI review pass

The package also builds a `dz-review` CLI for terminal review passes over the
same review annotations recognized by the extension:

```sh
dz-review review file.md
dz-review r file.md
```

Run `dz-review` without arguments to show help. To review annotations visible in
the current diff against `HEAD`, use `dz-review review` or its short alias
`dz-review r`.

For each custom review annotation or active conversation, the CLI shows the
block and prompts for an action. Annotation actions are:

- `n`: next;
- `a`: apply the annotation;
- `x`: cancel the annotation;
- `d`: delete the raw annotation block;
- `f`: stop reviewing the current file and move to the next file;
- `q`: quit immediately.

The hidden `p` action jumps to the next pending conversation. It is omitted from
the compact prompt but shown by `?`.

Conversation actions are:

- `n`: next;
- `d`: delete the conversation block;
- `o`: toggle a trailing `@me ok` reply;
- `r`: append an `@me` reply;
- `f`: stop reviewing the current file and move to the next file;
- `q`: quit immediately.

The hidden `p` action is also available from conversation prompts.

Inline conversations are displayed as multiline blocks in the terminal so each
`@agent`, `@me`, or bare `@` message is easier to read. On TTY output, headers,
roles, and annotation kinds use ANSI colors.

Color output defaults to the environment and TTY: `NO_COLOR=1` disables ANSI
colors, `FORCE_COLOR=1` enables them, and otherwise colors are enabled only
when stdout is a TTY. Use `--color always`, `--color never`, `--color auto`, or
`--no-color` to override the environment. When colors are enabled, review syntax
markers such as `{++`, `%...|`, and `++}` are dimmed separately from the
annotation content.

By default, each item also shows two source lines before the annotated line and
no source line after it. Configure that context with:

```sh
dz-review r --context 2:0 file.md
dz-review r -c 3 file.md
dz-review r --context-before 4 --context-after 1 file.md
```

`--context N` and `-c N` are shortcuts for `--context N:N`.

To process only conversations:

```sh
dz-review r --conversations file.md
dz-review r --open-conversations file.md
dz-review r --wip-conversations file.md
dz-review r --handled-conversations file.md
dz-review r --resolved-conversations file.md
dz-review r --pending-conversations file.md
```

Conversation status is based on the final review line:

- `open`: the last message is `@agent ...`;
- `wip`: the last message is `@` or `@me` with no text after it;
- `handled`: the last message is `@ ...` or `@me ...`, except `ok`;
- `resolved`: the last message is `@ ok`, `@me ok`, or `@me: ok`,
  case-insensitively.

The short filters `--open`, `--wip`, `--handled`, `--resolved`, and
`--pending` are also available. Short status filters apply to all review items:
non-conversation annotations such as additions, comments, and notes are treated
as `handled`. `--pending` is a shortcut for open or wip conversations.
`--ignore-closed-conversations` is kept as an alias for
`--pending-conversations`.

To focus on annotations visible in the current worktree diff against `HEAD`:

```sh
dz-review r --git file.md
dz-review review
dz-review r
```

This uses `git -c core.quotePath=false diff --no-color --unified=0 HEAD -- file.md`,
filters annotations whose current line range overlaps added lines in that diff,
and still edits the worktree file. Passing `core.quotePath=false` keeps paths
with accents or other Unicode characters readable and matchable. The internal
Git command also passes `--no-color` so inherited Git color settings cannot add
ANSI escapes to the diff parser input.

Useful combinations:

```sh
dz-review r --git --open-conversations file.md
dz-review r --git --pending file.md
dz-review list --git file.md
dz-review status --oneline file.md
dz-review status --short file.md
dz-review status file.md
dz-review st file.md
dz-review list
dz-review list file.md
dz-review l file.md
dz-review ls file.md
dz-review list --pending file.md
dz-review d
dz-review status --oneline --open-conversations file.md
dz-review status --oneline --pending-conversations file.md
dz-review status --pending file.md
dz-review status --oneline --resolved-conversations file.md
dz-review list --since 2026-06-16T17:35:35+02:00 file.md
dz-review list --since %1WzvP91W file.md
dz-review timestamp file.md
dz-review ts --iso file.md
dz-review ts --short file.md
dz-review ts -s -I file.md
dz-review ts -i -S file.md
dz-review ts --stdin -I
dz-review ts -o output.md file.md
dz-review ts -i file.md
dz-review now
dz-review now -I
dz-review now -S -d 2026-06-16T17:35:35+02:00
dz-review list --color always file.md
dz-review list --no-color file.md
```

`dz-review status --oneline` summarizes matching annotations in compact prose.
Without explicit files inside a Git worktree, it summarizes annotations visible
in the current worktree diff against `HEAD`. `dz-review status` without files
lists the files in that diff that contain matching annotations, with per-file
summaries:

```text
first.md: 1 conversation (1 open, 0 wip, 0 handled, 0 resolved); last @agent 2026-06-16T17:35:35+02:00
second.md: 1 addition
```

When present, `last` reports the newest timestamped review item: `@agent` or
`@me` for conversation messages, and `other` for non-conversation annotations.
Bare `@` messages are displayed as `@me`.

Use `dz-review status --short` for a denser ASCII format:

```text
file.md: 11/22 +4-2~2=7x2 - o 11:16:34
```

The `11/22` segment means 11 pending conversations out of 22 conversations.
The annotation segment uses `+` for additions, `-` for deletions, `~` for
substitutions, `=` for highlights, and `x` for comments. The last source is
`a` for `@agent`, `@` for `@me` or bare `@`, and `o` for other annotations.
Short timestamps are converted to the local timezone and omit the date when the
timestamp is today. The long `status` format keeps ISO timestamps.

`dz-review list file.md` prints the matching review items from the whole file
with source context and formatted conversation bodies, without editing files.
Without files, `dz-review list` prints matching review items from the current
worktree diff. `dz-review l` and `dz-review ls` are aliases for `list`.
`dz-review diff` and `dz-review d` are aliases for the Git diff form.

`dz-review timestamp file.md`, or its alias `dz-review ts file.md`, converts
existing timestamps to the selected format and adds missing timestamps to
`@agent`, `@me`, and bare `@` conversation messages, plus review annotations
such as `{++text++}`, using each file's modification time. It writes the
transformed content to stdout by default. Use `-o output.md` to write a new file
or `-i` / `--inline` to modify source files. Stdout and `-o` require exactly one
input file; use `-i` for multi-file updates. Use `-S` / `--short` or `-I` /
`--iso` to choose the output timestamp format; `--short` is the default.
Use `-s` / `--stdout` to force stdout output.
With no file and piped stdin, `dz-review ts` reads stdin and writes the
transformed content to stdout. Use `--stdin` to force stdin mode. `-i` /
`--inline` and `-o` / `--output` are invalid with stdin.
`--timestamp-format compact|short|iso` is kept for compatibility. `--since
TIMESTAMP` filters listings, status, and interactive passes to
timestamped items at or after the given ISO or compact timestamp. For
conversations, the last message timestamp is used.

`dz-review now` prints a timestamp for the current date in compact form. Use
`-I` / `--iso` for readable ISO output, or `-d DATE` / `--date DATE` to render a
specific ISO date, compact timestamp, or other date accepted by JavaScript
`Date`.

The CLI reads `.dz-review-ignore` from the current directory and skips matching
paths in all modes. The ignore file supports blank lines, `#` comments, simple
glob patterns, directory patterns such as `src/`, and negation with `!`.
